# Bug Analyzer Prompt

This is the prompt used with `/loop 5m` to run the bug analyzer.

## Usage

```bash
/loop 5m "$(cat analyzer-prompt.md)"
```

Or simply:

```bash
/loop 5m analyze-jira-bugs
```

---

## Prompt

Analyze Jira bugs with the `ib-ai-triage` label:

1. **Query Jira** via mcp-atlassian for issues with label `ib-ai-triage`, ordered by created date ascending, limit to 3 issues

2. If no issues found, report "No ib-ai-triage issues found" and exit

3. **Read repositories.md** to understand available repositories and their context this file also includes information about how the repos work together and how they provide different interfaces for the user

4. **For each issue (max 3):**
   
   a. Extract keywords from issue summary and description (error messages, component names, function names, stack traces)

   b. Look for possible duplicates in the following work item filter: https://redhat.atlassian.net/issues/?filter=111303
   
   c. Search all repositories for relevant code:
      - Use grep to find files containing keywords
      - Read relevant files to understand context
      - Follow imports and function calls if needed
      - Refine search based on findings
   
   d. Look to see if the bug may have been recently fixed.  its possible its no longer a bug.

   e. Perform medium-depth analysis:
      - Identify relevant files with line numbers
      - Trace likely execution path
      - Hypothesize root cause
      - Propose 2-3 potential fixes with trade-offs
      - Estimate effort (Small/Medium/Large with reasoning)
   
   f. Format analysis as markdown comment:
      ```markdown
      ## AI Analysis
      
      **Analyzed:** [current UTC timestamp]
      
      ### Relevant Files
      - `repo/path/to/file:line-range` - description
      
      ### Analysis
      [2-3 paragraphs explaining likely cause and execution path]
      
      ### Potential Fixes
      
      **Option 1: [Approach name]**
      - Description of fix
      - Trade-offs
      - Files affected
      
      **Option 2: [Alternative approach]**
      - Description
      - Trade-offs
      
      ### Effort Estimate
      **[Small/Medium/Large]** ([X-Y hours])
      - Reasoning for estimate
      
      ---
      *Generated by Claude Code Bug Analyzer*
      ```
   
   g. Post comment to Jira issue via mcp-atlassian
   
   h. Update labels via mcp-atlassian:
      - Remove `ib-ai-triage`
      - Add `ib-ai-analyzed`
   
   i. Report success: "Analyzed [ISSUE-KEY]: [summary]"

5. **Error Handling:**
   - If Jira query fails: report error and exit
   - If repository read fails: skip that repo, note in analysis
   - If analysis fails for an issue: leave labels unchanged, report error, continue to next issue
   - If MCP tools fail: report error and exit

6. Report summary: "Analyzed X issues this cycle"
